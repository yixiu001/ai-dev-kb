# 新手提示词包

> 用法：选择当前阶段，只复制对应提示词。不要一次把所有提示词发给 AI。

## 1. 第一次启动项目

```text
现在进入 DISCOVER 模式。

请先读取 AI_START.md、AI_DOCS.md、当前平台适配器和项目知识文件。

我的想法是：
<用自己的话描述>

请先不要写代码，不要选择技术栈，也不要增加功能。
请：
1. 用一句话复述项目；
2. 区分事实、建议、假设和缺失信息；
3. 每轮最多问 5 个关键问题；
4. 帮我填写 PROJECT.md；
5. 提出 V1 范围和明确非范围；
6. 等我批准。
```

## 2. 检查 AI 是否理解项目

```text
请先不要修改任何文件。
请说明：
1. 项目为谁解决什么问题；
2. 当前版本和阶段；
3. V1 做什么和不做什么；
4. 当前任务关联的需求；
5. 受影响的模块、数据和接口；
6. 明确禁止修改的内容；
7. 仍缺少或冲突的信息；
8. 完成后需要什么真实证据。
```

## 3. 生成 PRD

```text
现在进入 DEFINE 模式。
只完成 PRD.md，不选技术栈，不写代码。

请按顺序帮助我：
1. 确认用户角色和权限；
2. 走完 V1 核心用户旅程；
3. 补充失败、无权限、重复、取消和恢复流程；
4. 提取业务规则和状态机；
5. 为需求分配 RQ 编号；
6. 为每项核心需求写可观察的验收标准；
7. 明确非范围；
8. 生成草案并逐段等待我确认。
```

## 4. 选择技术方案

```text
现在进入 DESIGN 模式。
请读取已批准的 PROJECT.md 和 PRD.md。

我的能力和限制：
- 熟悉技术：<填写>
- 预算：<填写>
- 预计用户量：<填写>
- 部署偏好：<填写>
- 需要的外部能力：<填写>

请给出最多 3 套技术方案，比较开发难度、维护成本、初期费用、扩展能力、主要风险和退出难度。
不要为了未来可能的大量用户而过度架构。
等我选择后再填写 ARCHITECTURE.md。
```

## 5. 创建开发任务

```text
现在进入 IMPLEMENT 计划阶段，先不要修改代码。

本次目标：<填写>
关联需求：<填写 RQ ID>

请输出任务契约：
1. 用户结果；
2. 当前和期望行为；
3. 允许修改的模块和文件；
4. 禁止修改内容；
5. 数据、接口、权限和兼容性影响；
6. 测试列表；
7. 回退方式；
8. 预计修改规模。

任务必须足够小，可以完整审查和独立验证。
```

## 6. 批准实施

```text
我批准 <TASK ID>。
只允许在已批准范围内修改。
不要进行无关重构、全仓库格式化、依赖升级或额外功能开发。
发现其他问题时记录为风险或后续任务。
请在真实环境运行必要的构建和测试，并保留结果。
```

## 7. 独立验证

```text
现在进入 VERIFY 模式。
请独立审查 <TASK ID>，不要继续增加功能。

请读取需求、业务规则、实际 Diff、测试和运行结果。
先生成测试矩阵，覆盖：
- 正常流程；
- 错误输入；
- 无权限和跨用户访问；
- 重复操作和幂等；
- 超时、重试和外部服务失败；
- 旧功能回归。

请逐项提供真实执行结果，并判断最高可信状态。
```

## 8. 分析错误

```text
现在进入 VERIFY 模式，不要继续扩大修改。

当前版本：<填写>
最后正常版本：<填写>
执行命令：<填写>
完整错误日志：<粘贴>
预期行为：<填写>
实际行为：<填写>
最近修改：<填写>

请：
1. 对问题分类；
2. 给出概率最高的 3 个根因；
3. 为每个根因给出最小验证方法；
4. 本轮只验证一个假设；
5. 声明禁止修改范围；
6. 说明失败时怎样回退。
```

## 9. 发布前检查

```text
现在进入 RELEASE 规划模式，不执行生产修改。

请读取 RELEASE.md、当前版本、全部验证证据和已知风险。
请生成上线前检查结果：
- 已通过项；
- 未通过项；
- 缺少证据项；
- 阻塞发布项；
- 数据迁移风险；
- 权限和安全风险；
- 监控与回退准备；
- 是否建议发布。

只要存在阻塞项，就不要建议继续发布。
```

## 10. 生成发布 Runbook

```text
请为版本 <版本 / Commit> 生成发布 Runbook。
每一步必须包含：
- 执行人或工具；
- 命令或操作；
- 预期结果；
- 证据；
- 停止条件；
- 回退操作。

先输出草案，等待我批准，不直接执行生产操作。
```

## 11. 新对话接手项目

```text
请先读取 AI_START.md、AI_DOCS.md、当前平台适配器，以及本项目正式文件。

在开始工作前，请复述：
1. 项目目标；
2. 当前版本和阶段；
3. 已批准范围；
4. 当前任务；
5. 关键决定和禁止事项；
6. 已有证据和证据缺口；
7. 你与上一个平台或会话在能力和权限上的差异。

我确认理解正确后再继续。
```

## 12. 结束任务并交接

```text
请结束本次任务并输出交接记录：
1. 实际修改；
2. 未修改范围；
3. 执行命令；
4. 通过和失败的检查；
5. Commit、PR、日志或其他证据；
6. 当前可信状态；
7. 剩余风险；
8. 需要更新的项目文件；
9. 下一项唯一任务。

请把真实状态写回 PLAN_AND_STATE.md，把决定、风险和证据写回对应文件。
```
