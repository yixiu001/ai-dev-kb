# 第 1 步：我用 30 分钟准备好开发环境

> 面向：第一次开始项目的用户

## 这一步完成后，我会得到什么

完成后，我应该拥有：

- 一个专门用于当前项目的 AI 工作空间；
- 一个 GitHub 仓库；
- 一套项目知识文件；
- 一个能够运行代码的环境；
- 一个明确的“资料放哪里、代码放哪里、任务在哪里继续”的规则。

## 开始前我需要准备

- 一个可以使用的 AI 平台账号；
- 一个 GitHub 账号；
- 一台可以打开浏览器的电脑；
- 如果我要在本地开发：安装 Git 和 VS Code；
- 如果我暂时不想配置本地环境：准备一个支持运行代码的云端开发环境。

## 第 1 个操作：创建 AI 项目空间

以 ChatGPT 网页端为例：

1. 打开 ChatGPT；
2. 新建一个 Project；
3. 项目名称使用真实产品名称，不使用“测试”“随便做做”；
4. 在项目说明中写入下面内容；
5. 把本知识库的 `AI_START.md` 和 `AI_DOCS.md` 加入项目，或确保 AI 可以读取仓库。

我可以把下面这段放进项目说明：

```text
这个 Project 用于开发一个正式软件产品。
所有重要结论必须写入项目正式文件，不能只保存在聊天中。
开始任务前先读取 AI_START.md、AI_DOCS.md、当前平台适配器和项目状态。
用户提问不等于授权修改。
没有真实运行和测试证据时，不得声称功能已经完成。
请始终使用中文与我沟通。
```

## 第 2 个操作：创建 GitHub 仓库

1. 登录 GitHub；
2. 点击新建仓库；
3. 仓库名称使用项目英文名，例如 `creator-insight`；
4. 选择 Public 或 Private；
5. 不要把密码、Token、Cookie 或真实 `.env` 提交到公开仓库；
6. 创建仓库后记录仓库地址。

### 我应该选择 Public 还是 Private

| 情况 | 建议 |
|---|---|
| 我要公开源码 | Public |
| 项目包含商业逻辑、客户资料或未公开产品 | Private |
| 我还不确定 | 先 Private，后续再决定 |

## 第 3 个操作：复制项目知识包

把 `05-project-template/` 中的六个文件复制到新项目的 `docs/project/` 目录：

```text
my-project/
├─ docs/
│  └─ project/
│     ├─ PROJECT.md
│     ├─ PRD.md
│     ├─ ARCHITECTURE.md
│     ├─ PLAN_AND_STATE.md
│     ├─ DECISIONS_RISKS_EVIDENCE.md
│     └─ RELEASE.md
└─ README.md
```

如果我不会复制文件，我可以让已经连接 GitHub 的 AI 帮我创建；如果 AI 没有写入权限，我就手动在 GitHub 网页中创建这些文件。

## 第 4 个操作：确认代码在哪里运行

我必须选择一种真实运行方式。

### 方案 A：本地开发

适合我愿意安装工具，并希望长期维护项目。

最低需要：

- Git；
- VS Code；
- 项目技术栈需要的运行时，例如 Node.js 或 Python；
- 终端。

### 方案 B：云端开发

适合我暂时不想处理本地安装问题。

我需要确认云端环境能否：

- 保存文件；
- 运行终端命令；
- 安装依赖；
- 查看日志；
- 连接 GitHub；
- 导出或部署项目。

## 第 5 个操作：让 AI 检查我的准备结果

我把下面内容发给 AI：

```text
现在进入 ANSWER 模式，不要修改任何文件。
请检查我的项目初始化是否完整。

AI 工作空间名称：<填写>
GitHub 仓库地址：<填写>
代码运行环境：本地 / 云端
项目知识文件位置：<填写>

请输出：
1. 已准备完成的内容；
2. 缺少的内容；
3. 可能存在的安全风险；
4. 我下一步只需要做的一件事。
```

## AI 应该返回什么

AI 应该给我一张简短检查表，而不是开始设计产品或写代码：

| 检查项 | 状态 | 说明 |
|---|---|---|
| AI 工作空间 | 已完成 | 已有独立 Project |
| GitHub 仓库 | 已完成 | 仓库地址可访问 |
| 项目知识文件 | 缺失 | 尚未创建六个文件 |
| 运行环境 | 已完成 | 本地 Node.js 可用 |
| 敏感信息保护 | 待确认 | 需要检查 `.gitignore` |

## 我怎样判断这一步完成

- [ ] 我有独立 AI 项目空间；
- [ ] 我有 GitHub 仓库；
- [ ] 六个项目知识文件已经存在；
- [ ] 我知道代码在哪里真实运行；
- [ ] 我没有把密钥放入仓库；
- [ ] AI 已经检查并指出下一步。

## 卡住时怎么办

### 我不会使用 GitHub

我先只完成仓库创建和文件上传，不急着学习分支、PR 和 Actions。后续需要时再学。

### 我不会安装开发环境

我先选择云端环境完成第一个项目；等项目方向稳定后，再迁移到本地。

### AI 说它已经创建了文件，但我看不到

我要求它提供真实仓库路径和 Commit；如果没有，就说明只是生成了内容，并没有实际写入。
