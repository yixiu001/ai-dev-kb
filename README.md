# AI Dev KB

一套面向普通用户、由 ChatGPT 驱动的**复杂软件产品开发知识库**。

它不是提示词合集，也不是简单脚本教程。它用于帮助用户把一个产品想法推进为能够服务真实用户、支持持续迭代的正式软件系统。

## 适用项目

- SaaS、Web 平台和移动应用
- AI 应用与自动化平台
- 包含用户、权限、数据库、支付和外部 API 的产品
- 需要面向大量用户、长期维护和持续发布的项目

## 本知识库解决什么

AI 编程在大型项目中的主要问题不是“写不出代码”，而是：

- 读错或遗漏上下文
- 忘记早期决定，产生架构漂移
- 擅自扩大修改范围
- 大改动难以审查和合并
- 测试不足，却错误声称已经完成
- 原型能运行，生产环境却失败
- 登录存在但授权和数据隔离错误
- 多个 AI 同时开发时互相覆盖
- 上线后缺少日志、监控、回滚和运营能力

V1 用项目事实、阶段门禁、任务契约、需求追踪、架构边界、质量门禁和生产运行手册解决这些问题。

## 普通用户如何使用

你不需要自己写完所有专业文档。让 ChatGPT 按以下顺序采访、整理并等待你的确认：

1. 复制 `05-project-template/` 到项目仓库或项目资料目录。
2. 让 AI 先读取 `AI_START.md`。
3. 使用 `PROJECT.md` 明确市场、用户、价值、商业目标和 V1 边界。
4. 使用 `PRD.md` 建立角色、流程、业务规则、状态、权限和验收标准。
5. 使用 `ARCHITECTURE.md` 建立领域、模块、数据、接口、安全和规模设计。
6. 使用 `PLAN_AND_STATE.md` 把开发拆成小任务并持续记录真实状态。
7. 使用 `DECISIONS_RISKS_EVIDENCE.md` 保存决定、风险和验证证据。
8. 每次只执行一个明确任务，经过质量门禁后再合并。
9. 使用 `RELEASE.md` 完成部署、线上验证、监控和回滚准备。

## 核心原则

- 项目文档保存意图，代码仓库保存实现，测试与生产证据保存事实。
- AI 不依赖“记住整个项目”，而依赖按任务提取最小且最新的上下文。
- 任何复杂改动必须先分析影响，再实施，再验证。
- “已生成、已执行、已测试、已发布、已验证”是五种不同状态。
- 普通用户可以决定产品方向，但安全、支付、数据迁移和生产发布必须经过硬性门禁。
- AI 可以加速工程，但不能替代真实运行、独立审查和生产责任。
