# 复杂项目知识包模板

复制本目录到具体项目中。用户不需要一次填完，由 AI 按阶段采访并维护。

## 文件

| 文件 | 作用 |
|---|---|
| `PROJECT.md` | 市场、用户、价值、商业目标和范围 |
| `PRD.md` | 角色、流程、业务规则、状态、权限和验收 |
| `ARCHITECTURE.md` | 领域、模块、数据、接口、安全和规模设计 |
| `PLAN_AND_STATE.md` | 里程碑、任务契约和当前真实状态 |
| `DECISIONS_RISKS_EVIDENCE.md` | 决定、风险、假设和验证证据 |
| `RELEASE.md` | 环境、迁移、发布、监控和回滚 |

## AI 读取原则

- 先读取项目和当前状态，再根据任务读取相关章节。
- 不复制所有文档到每个提示中。
- 历史聊天不是事实来源。
- 任何状态变化必须有证据。
- 文档与代码冲突时明确报告。

## 更新责任

- 产品范围变化：更新 `PROJECT.md` 和 `PRD.md`
- 架构与接口变化：更新 `ARCHITECTURE.md` 和决定记录
- 任务完成或阻塞：更新 `PLAN_AND_STATE.md`
- 新假设、风险、测试或部署结果：更新证据文件
- 发布与事故：更新 `RELEASE.md`
