# GPT-5.5 模式选择

更新日期：2026-07-07。模型名称、额度和可用性会变化，使用前以 ChatGPT 当前界面和 OpenAI 官方说明为准。

## 选择原则

| 模式 | 适合任务 |
|---|---|
| Instant | 简单问答、文档调整、短代码、明确的小修改 |
| Medium | 常规需求分析、调试、功能实现、方案比较 |
| High | 复杂架构、跨文件问题、疑难调试、重要审查 |
| Extra High | 高复杂度且错误成本高的推理任务，取决于套餐 |
| Pro | 最困难任务、长流程研究和高要求分析，取决于套餐 |

## 开发中的推荐用法

- 日常整理和小任务：Instant。
- 正式需求、架构、编码和调试：Medium。
- 多模块设计、难复现错误、上线前审查：High。
- 重要技术决策或前几种模式表现不足：更高推理级别或 Pro。

## 不应只按“最强模型”选择

模型越强不代表每个任务都更合适。还要考虑：

- 任务复杂度
- 响应速度
- 使用额度
- 上下文长度
- 是否需要工具
- 错误成本

## 工具支持

根据 OpenAI 当前说明：

- GPT-5.5 Thinking 支持 ChatGPT 中全部可用工具。
- GPT-5.5 Instant 支持当前工具，但不支持 Canvas。
- Plus 用户通常可选择 Instant、Medium 和 High；更高选项依套餐而定。

## 降级策略

1. 高级模式不可用时，先缩小任务范围。
2. 将长任务拆成分析、实施、验证三步。
3. 提供更完整项目文档，减少模型猜测。
4. 对关键结论使用第二次审查，而不是盲目提高推理级别。
