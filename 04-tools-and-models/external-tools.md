# 外部工具与服务

外部工具用于弥补 ChatGPT 网页端不能天然访问本地代码、仓库、服务器和业务系统的边界。

## 常见类型

| 类型 | 用途 |
|---|---|
| GitHub | 读取仓库、提交文件、管理 Issue 和 PR、检查 CI |
| 云盘 | 读取项目资料和交付文件 |
| 邮件与日历 | 沟通、通知和安排工作 |
| MCP / 自定义 App | 连接内部 API、数据库或业务工具 |
| 部署平台 | 构建、发布、日志和环境变量管理 |
| 浏览器或计算机操作 | 网页测试和必须通过界面完成的操作 |

## 调用前

- 确认工具是否已连接。
- 确认当前账号和权限。
- 明确读取还是写入。
- 评估是否涉及费用、删除、公开发布或敏感数据。
- 写入操作应说明对象、内容和影响。

## 调用后

- 读取返回结果。
- 验证对象、分支、文件、环境或收件人是否正确。
- 不把“请求已发送”自动等同于“目标系统已完成”。
- 将重要结果写入项目状态或部署记录。

## 权限原则

- 默认最小权限。
- 读取优先于写入。
- 修改优先于删除。
- 测试环境优先于生产环境。
- 高风险操作必须获得用户明确确认。

## 外部调用失败

1. 保存错误信息。
2. 判断是权限、参数、网络、额度还是服务故障。
3. 不进行无意义重复调用。
4. 提供用户可以执行的最小修复步骤。
5. 修复后重新读取目标状态验证。
