# GitHub 仓库与文件管理

> 面向：需要建立和维护正式代码仓库的用户

## 一个仓库应该保存什么

仓库通常保存：

- 源代码；
- 项目配置；
- 测试；
- 数据库迁移；
- 文档；
- 自动化工作流；
- `.gitignore`；
- `.env.example`；
- License；
- 发布记录。

仓库不应该保存：

- 密码、Token、Cookie、私钥；
- 真实 `.env`；
- 用户隐私数据；
- 生产数据库备份；
- `node_modules`、缓存和构建临时文件；
- 大型视频和原始素材，除非使用合适的存储方案。

## Public 和 Private

| 类型 | 谁可以看 | 适合什么 |
|---|---|---|
| Public | 任何人 | 开源项目、公开课程、作品集 |
| Private | 只有授权成员 | 商业项目、客户项目、未公开产品 |
| Internal | 企业成员 | Enterprise 内部共享 |

改变仓库可见性可能影响 Fork、Pages、Actions、规则和外部链接，执行前应阅读 GitHub 的提示。

## 创建仓库时的基本选择

- Repository name：使用稳定、易懂的英文名；
- Description：一句话说明仓库用途；
- Visibility：不确定时先 Private；
- README：建议创建；
- `.gitignore`：选择真实技术栈；
- License：只有准备明确授权他人使用时再选择。

## README 应该写什么

一个正式项目 README 至少包含：

```text
项目是什么
→ 当前状态
→ 技术栈
→ 本地运行方法
→ 环境变量
→ 测试方法
→ 部署说明入口
→ 项目文档入口
```

README 不应堆积完整 PRD 和架构细节，而应链接到对应文档。

## `.gitignore` 的作用

`.gitignore` 告诉 Git 不追踪哪些文件，例如：

```gitignore
.env
.env.*
!.env.example
node_modules/
dist/
.venv/
__pycache__/
*.log
.DS_Store
.vscode/settings.json
```

注意：如果敏感文件已经被 Commit，后来加入 `.gitignore` 不会自动从历史中删除。

## `.env.example`

`.env.example` 只保存变量名和示例，不保存真实值：

```env
DATABASE_URL=
OPENAI_API_KEY=
APP_BASE_URL=http://localhost:3000
```

## License 怎样选择

| 情况 | 选择 |
|---|---|
| 私有商业项目 | 可以不添加公开 License |
| 希望别人自由使用和修改 | MIT、Apache-2.0 等 |
| 希望衍生项目也开源 | GPL 系列 |
| 不确定法律影响 | 不要让 AI 随机选择，先确认用途 |

没有 License 的公开代码不等于别人可以随意复制和商用。

## Tags 和 Releases

- Tag：指向一个特定 Commit 的版本标记，例如 `v1.0.0`；
- Release：基于 Tag 创建的正式发布页面，可以包含说明和构建文件。

推荐版本：

```text
v1.0.0  正式首版
v1.1.0  增加向后兼容功能
v1.1.1  修复缺陷
```

## 大文件和 Git LFS

Git 不适合频繁修改的大型二进制文件。

适合 Git LFS 的内容可能包括：

- 模型文件；
- 大型设计源文件；
- 音视频；
- 大型数据集；
- 二进制安装包。

Git LFS 有存储和带宽配额，超过后可能收费。产品用户上传文件通常应放对象存储，而不是 Git 仓库。

## Archive 和 Delete

- Archive：仓库变为只读，适合停止维护但保留历史；
- Delete：删除仓库，高风险；
- Transfer：把仓库转移给其他个人或组织。

删除前应确认：

- 已有本地镜像或备份；
- Actions、Packages、Releases 和 Pages 的影响；
- 域名和外部服务是否仍引用仓库；
- 团队成员是否知情。

## 推荐目录

```text
my-project/
├─ .github/
│  ├─ workflows/
│  ├─ ISSUE_TEMPLATE/
│  └─ pull_request_template.md
├─ docs/
│  └─ project/
├─ src/
├─ tests/
├─ scripts/
├─ .env.example
├─ .gitignore
├─ LICENSE
└─ README.md
```

## 仓库初始化检查

- [ ] 可见性正确；
- [ ] README 可指导本地启动；
- [ ] `.gitignore` 符合技术栈；
- [ ] 没有真实密钥；
- [ ] 项目文档有固定位置；
- [ ] 测试和工作流目录明确；
- [ ] License 选择符合真实用途；
- [ ] 大文件有单独策略。
