# AI 复杂项目协作模型

## 1. AI 的正确定位

ChatGPT 在大型项目中应同时承担分析师、设计助手、实施助手和审查助手，但不能在同一轮中无边界切换角色。

每项任务必须经过：

```text
理解 → 计划 → 批准 → 实施 → 验证 → 记录
```

不允许从模糊需求直接跳到大规模代码修改。

## 2. 人与 AI 的职责

### 用户负责

- 业务目标、预算和风险承受能力
- V1 范围和关键优先级
- 产品、架构和发布批准
- 账号、权限和真实环境
- 对真实用户和生产结果负责

### AI 负责

- 补齐问题和信息缺口
- 将自然语言转换为可追踪的项目资产
- 分析方案、影响、风险和替代选择
- 生成受控的小型实现
- 设计和执行可用工具支持的验证
- 保持项目状态和证据可追溯

## 3. 阶段门禁

| 阶段 | 必须通过的门禁 |
|---|---|
| 产品发现 | 用户、问题、市场假设、价值和成功指标明确 |
| 产品定义 | 角色、流程、规则、权限、异常和验收标准明确 |
| 系统设计 | 模块、数据、接口、安全、规模和部署路径明确 |
| 实施 | 任务可独立验证、范围小、依赖清楚 |
| 验证 | 自动测试、业务验收、安全检查和证据通过 |
| 发布 | 环境、迁移、回滚、监控和线上冒烟测试就绪 |
| 运营 | 指标、告警、事故流程、用户反馈和迭代入口存在 |

## 4. 双层审查

重大决定和高风险实现不得只由同一个对话自己提出并批准。

推荐：

- 第一轮 AI 负责设计或实现。
- 第二轮使用新对话、不同模型或明确的审查角色寻找反例。
- 确定性工具负责构建、测试、扫描和数据验证。
- 用户批准最终取舍。

## 5. 普通用户保护机制

以下领域不能只凭界面“看起来能用”验收：

- 身份认证与授权
- 多租户数据隔离
- 支付、订阅、退款和 Webhook
- 数据迁移、删除和备份
- 文件上传与公开访问
- 管理员和高权限操作
- 生产环境发布与域名配置

这些领域必须有明确设计、自动检查和真实环境验证。
