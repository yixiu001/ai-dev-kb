## 我边做边使用的两个工具

- [`新手项目工作表`](./00-user-guide/workbook/新手项目工作表.md)：记录我的项目答案和完成状态；
- [`新手提示词包`](./00-user-guide/prompts/新手提示词包.md)：按当前阶段复制给 AI。

## 我会建立的六个正式项目文件

```text
PROJECT.md
PRD.md
ARCHITECTURE.md
PLAN_AND_STATE.md
DECISIONS_RISKS_EVIDENCE.md
RELEASE.md
```

我不需要一开始自己写完这些文件。AI 会通过提问帮助我填写，我负责确认内容是否真实、是否符合我的目标。

## 我与 AI 怎样分工

| 我负责 | AI 负责 |
|---|---|
| 描述真实问题 | 整理成结构化项目定义 |
| 决定目标用户和范围 | 提出选项、影响和风险 |
| 批准需求和架构 | 维护正式项目文档 |
| 提供账号、权限和真实反馈 | 生成和修改代码 |
| 决定是否发布 | 设计并执行可用的测试 |
| 对真实用户和业务负责 | 保存状态、风险和证据 |

## 我开始第一个会话时可以直接复制

```text
我正在按照 AI Dev KB 的新手实战路线启动一个正式项目。

请先读取：
1. AI_START.md；
2. AI_DOCS.md；
3. NOVICE_GUIDANCE_PROTOCOL.md；
4. 当前平台适配器；
5. 我的项目正式文件。

现在不要写代码。
请先告诉我当前应该完成实战路线中的哪一步，并只引导我完成一个可检查的操作。
每次请说明：
- 我在哪里操作；
- 我需要复制或填写什么；
- 我应该看到什么结果；
- 我怎样判断成功；
- 失败时我应该把什么信息发给你。
```

## 一个重要规则

AI 给我的文字不是完成证据。

- 生成了代码：`GENERATED`
- 真实运行了：`EXECUTED`
- 需求和测试都通过：`VERIFIED`
- 发布到生产环境：`RELEASED`
- 经过真实用户和监控观察：`OBSERVED`

我会要求 AI 明确当前属于哪一个状态。
