# 阶段 3：技术设计

## 用户提供

- 已确认需求和产品设计
- 现有代码或技术环境
- 预算、部署平台和维护能力
- 必须接入的外部服务

## AI 完成

- 选择满足 V1 的最简单可维护技术栈。
- 划分前端、后端、数据库和外部服务边界。
- 设计核心数据结构和接口。
- 说明认证、权限、安全和错误处理。
- 给出本地开发、测试和部署方式。
- 明确关键风险和需要验证的技术假设。

## 输出

- `ARCHITECTURE.md`
- 模块和数据关系
- API 或接口约定
- 环境与依赖清单
- 风险验证计划

## 完成条件

- 每项核心需求都有实现位置。
- 技术栈与用户能力、预算匹配。
- 外部依赖和权限要求明确。
- 部署路径明确。
- 用户批准进入任务拆分和开发。
