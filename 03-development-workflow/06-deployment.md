# 阶段 6：部署上线

## 用户提供

- 部署平台与账号权限
- 域名、环境变量和生产配置
- 预算与服务限制
- 上线时间和回滚要求

## AI 完成

- 检查构建、环境变量和生产配置。
- 提供可重复的部署步骤。
- 说明数据库迁移、存储和外部服务配置。
- 准备上线后验证和回滚方案。
- 根据真实部署结果定位失败原因。

## 上线顺序

```text
备份与检查
→ 构建
→ 部署
→ 配置环境变量和域名
→ 运行迁移
→ 线上冒烟测试
→ 监控错误
→ 记录版本
```

## 输出

- 部署记录
- 生产访问地址
- 配置清单（不含密钥值）
- 线上验证结果
- 回滚方法
- 当前版本与已知问题

## 完成条件

- 用户可以从公开或目标网络访问。
- 核心流程在线上真实通过。
- 日志和监控可用。
- 敏感信息未泄露。
- 回滚路径明确。
