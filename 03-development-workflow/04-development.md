# 阶段 4：开发

## 用户提供

- 可访问的代码仓库或项目文件
- 开发和测试环境
- 必需账号与外部服务配置
- 当前任务的明确批准

## AI 完成

- 将架构拆成小而可验证的任务。
- 按依赖顺序逐项实施。
- 每次修改说明文件、目的和影响。
- 为关键逻辑编写测试或验证步骤。
- 遇到需求或架构冲突时停止并报告。

## 推荐节奏

```text
选择一个任务
→ 读取相关代码
→ 说明修改方案
→ 实施最小改动
→ 运行或提供测试
→ 记录结果
→ 更新任务状态
```

## 输出

- 代码与配置
- `TASKS.md` 更新
- 开发和验证说明
- 新决定写入 `DECISIONS.md`

## 完成条件

- V1 核心功能均已实现。
- 项目可以在目标环境启动。
- 不存在已知阻塞错误。
- 关键功能有测试方法。
- 可以进入系统测试。
