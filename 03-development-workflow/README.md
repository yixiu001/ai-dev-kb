# 从想法到上线的六阶段流程

```text
想法与需求 → 产品设计 → 技术设计 → 开发 → 测试 → 部署上线
```

每个阶段都必须明确：

1. 用户提供什么
2. AI 完成什么
3. 输出什么文档或成果
4. 满足什么条件才能进入下一阶段

## 阶段门槛

| 阶段 | 必要输出 | 进入下一阶段的条件 |
|---|---|---|
| 想法与需求 | 项目简报、需求、验收标准 | 用户确认做什么和不做什么 |
| 产品设计 | 用户流程、页面、状态与异常 | 核心使用流程可闭环 |
| 技术设计 | 架构、数据、接口、部署方案 | 技术方案可实施且风险明确 |
| 开发 | 可运行代码、配置、开发记录 | 核心功能完成并能进入测试 |
| 测试 | 测试结果、问题清单 | 阻塞问题解决，验收标准通过 |
| 部署 | 可访问服务、部署记录、回滚方法 | 线上验证通过 |

不得为了追求速度跳过需求确认、真实测试或上线验证。
