# 阶段 5：测试

## 用户提供

- 可运行版本
- 测试环境和必要账号
- 已确认验收标准
- 真实设备或浏览器条件

## AI 完成

- 根据需求生成测试场景。
- 覆盖主流程、异常、权限和边界情况。
- 分析日志、错误和失败原因。
- 将问题按阻塞、严重、一般分类。
- 修复后安排回归测试。

## 最低测试范围

- 启动与基础配置
- 注册、登录和权限（如适用）
- 核心业务流程
- 空数据和错误输入
- 外部 API 失败
- 刷新、重复提交和网络中断
- 安全与敏感数据
- 关键浏览器或设备

## 输出

- 测试结果
- 缺陷与修复记录
- 未覆盖风险
- 上线建议

## 完成条件

- 所有核心验收标准通过。
- 阻塞和严重问题已解决。
- 已完成必要回归测试。
- 剩余风险被用户接受。
