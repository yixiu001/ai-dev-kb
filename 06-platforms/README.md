# AI 平台适配层

本目录只记录不同 AI 开发平台的执行差异，不复制产品、架构、安全、测试和发布等通用规则。

## 平台适配器必须回答

1. 平台如何保存和读取项目上下文。
2. 能否直接读取代码仓库和本地文件。
3. 能否运行终端、测试、构建和部署命令。
4. 能否进行网页搜索和读取最新资料。
5. 能否连接 GitHub、云盘、数据库和其他外部工具。
6. 哪些操作需要用户确认。
7. 平台的上下文、权限、会话和执行边界是什么。
8. 平台能够为任务状态提供什么级别的真实证据。
9. 复杂项目应如何拆分会话、Agent、分支或 Worktree。
10. 平台变化时如何检查适配器是否过期。

## 核心约束

- 适配器不得修改通用项目状态定义。
- 适配器不得降低安全、审查、测试或发布门禁。
- 平台声称支持某项能力，不代表当前账号、套餐或工作区一定可用。
- 开始任务时必须先检查实际可用工具和权限。
- 平台功能会变化，适配器必须标注最后核验日期和官方来源。
- 平台不可用的能力应交给外部工具或人工完成，不得伪造结果。

## 选择平台

选择标准不是“哪个 AI 最强”，而是当前任务需要什么：

| 任务需求 | 重点检查的平台能力 |
|---|---|
| 产品与架构研究 | 长上下文、网页研究、文件分析、引用 |
| 仓库级开发 | 代码检索、Diff、终端、测试、分支操作 |
| UI 与原型 | 图像理解、设计工具、浏览器预览 |
| 生产发布 | 云平台连接、Secret 管理、日志和确认机制 |
| 多 Agent 并行 | 任务隔离、分支/Worktree、共享状态和集成能力 |

一个复杂项目可以组合多个平台，但 `05-project-template/` 中的项目事实和状态必须保持唯一。
