# 发布与生产运营

## 1. 环境矩阵

为本地、测试、预发布和生产记录：

- 运行时和依赖版本
- 构建与启动命令
- 环境变量名称和来源
- 数据库、存储和队列
- 外部服务与回调地址
- 网络、域名和权限
- 数据规模和模拟方式

构建产物应尽量在各环境一致，只替换配置。

## 2. 发布准备

- 版本和 Commit 固定
- 数据库备份完成
- 迁移脚本在预发布验证
- 配置和 Secret 检查
- 发布说明和已知风险
- Feature Flag 或灰度策略
- 回滚版本和回滚命令
- 监控看板和告警已就绪

## 3. 发布步骤

```text
冻结版本
→ 备份
→ 构建并扫描
→ 部署
→ 运行迁移
→ 健康检查
→ 关键用户旅程冒烟测试
→ 观察日志与指标
→ 扩大流量
→ 记录结果
```

部署命令成功不等于发布成功。必须完成线上验证。

## 4. 生产事故

事故流程：

1. 确认影响和严重度
2. 停止继续发布
3. 保护数据和降低影响
4. 回滚、关闭功能或降级
5. 保存日志和时间线
6. 修复并验证
7. 编写复盘和预防任务

事故期间不得让多个 Agent 无协调地同时修改生产问题。

## 5. Day-2 运营

上线后持续维护：

- 错误率、延迟、资源和第三方服务
- 激活、留存、转化和付费
- 用户反馈和客服问题
- 数据备份和恢复演练
- 依赖、证书和密钥轮换
- 成本异常
- 安全事件和滥用

## 6. 发布完成定义

只有满足以下条件才能标记 `OBSERVED`：

- 生产地址和核心旅程可用
- 用户数据隔离和权限真实通过
- 日志与指标没有持续异常
- 真实外部服务成功和失败路径已验证
- 回滚可用
- 已经过定义的观察窗口
