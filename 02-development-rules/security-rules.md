# 安全规则

## 禁止进入仓库的内容

- API Key、Access Token、密码
- Cookie、Session、私钥、证书
- 真实 `.env` 文件
- 数据库连接凭证
- 用户隐私数据
- 生产环境机密配置

## 正确做法

- 使用环境变量。
- 提供 `.env.example`，只写变量名和示例格式。
- 在 `.gitignore` 中排除敏感文件。
- 使用最小权限原则。
- 测试和生产环境分离。
- 日志中隐藏密码、Token 和个人信息。

## 外部操作

以下操作执行前必须得到用户明确授权：

- 发送邮件或消息
- 创建、修改或删除云端数据
- 发布、部署或回滚生产服务
- 发起付费操作
- 修改权限、域名、支付或账号配置
- 删除仓库、分支、文件或数据库记录

## 依赖与外部代码

- 优先使用官方文档和可信来源。
- 检查依赖是否维护、许可证是否合适。
- 不直接运行来历不明的脚本。
- 安装命令和自动化脚本应先解释其影响。

## 发现泄露时

1. 停止继续传播。
2. 撤销或轮换凭证。
3. 从当前文件和 Git 历史中清除。
4. 检查是否被使用。
5. 记录修复和后续防护措施。
